## Design Context

### Users
Makerspace members (Hebewerk e.V. Eberswalde) using the interface on-site — at a shared screen in the workshop or on their phone while standing at a machine. They're makers: hands-on, technical, comfortable with density. Speed matters — they're mid-project, checking a status or starting a booking fast. Admins also use it for configuration and management.

### Brand Personality
**Technical · Industrial · Maker-proud**

The interface should feel like it was built by hackers for hackers — monospace text, dark theme, warm amber accents (workshop lighting, not neon). Dense information is fine. It should evoke the feeling of a control terminal for physical machinery, not a SaaS dashboard.

### Aesthetic Direction
- **Dark-mode first** with a warm charcoal palette (browns, not blue-grays)
- **Monospace typography** (JetBrains Mono) for body, Bebas Neue for display
- **Amber as primary accent** — machine-ready, high-visibility, industrial
- **Minimal decoration** — no glassmorphism, no gradients, no rounded softness. 2px border-radius max.
- **Dense, information-rich layouts** — makers want data, not whitespace

**Anti-references**: Generic SaaS dashboards, AI-generated dark-UI with purple/cyan gradients, overly friendly/rounded interfaces.

### Design Principles
1. **Information density over breathing room** — makers want status at a glance
2. **Color carries meaning** — amber/green/red/dim map to machine states; never use these decoratively
3. **Monospace is identity** — JetBrains Mono is intentional, keep it consistent
4. **Warm, not cold** — browns not blue-grays; industrial warmth, not corporate coldness
5. **Accessibility: WCAG AA minimum** — contrast ratios, keyboard navigation, semantic HTML
